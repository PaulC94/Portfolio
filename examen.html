E4 Coefficient 4

L'épreuve E4 est une épreuve écrite de 4 heures portant sur un cas pratique. Elle évalue la capacité du candidat à analyser une situation d'entreprise et à proposer des solutions pour le support, la mise à disposition de services informatiques et la cybersécurité. Le sujet est basé sur un contexte organisationnel fourni le jour de l'épreuve.

Compétences travaillées pour l'E4

Gestion du patrimoine informatique

Inventaire et gestion d'un parc de 300 postes via GLPI. Rédaction d'un PRA/PCA, gestion des sauvegardes MySQL, charte informatique. Travail réalisé lors du stage à la Mairie.

GLPIPRA/PCASauvegardesCharte IT

Support de niveau 2 via ticketing GLPI. Résolution d'incidents réseau (switch HP en panne), système (disque serveur saturé) et applicatif (logiciel comptable ralenti). Stage Mairie + EXTIA.

// 03 — Épreuve E6

Parcours de
compétences.

E6 Coefficient 4

L'épreuve E6 porte sur la cybersécurité des services informatiques et se divise en deux parties : client léger (applications web) et client lourd (applications desktop). Le candidat doit démontrer ses compétences dans le développement et la sécurisation de ces deux types d'applications.

Projet E6 — Muni-Prospection LAcarel

Application de prospection commerciale pour la société LAcarel, développée en deux versions : un client léger (web) et un client lourd (desktop). Les deux versions partagent la même base de données MySQL et permettent de gérer les communes, élus et saisies de prospection.

CL Client Léger · Web

Muni-Prospection — Version Web (PHP)

Application web en PHP 8.3 avec Bootstrap 5.3. Connexion via PDO avec Singleton, gestion de sessions et verrouillage pessimiste SQL pour éviter les conflits de saisie. Dashboard avec KPI, filtres, pagination.

PHP 8.3MySQL / PDOBootstrap 5.3JavaScriptSessions
Fichiers principaux :
  • config.php — Singleton PDO, gestion sessions, timeout verrous
  • index.php — Dashboard avec cartes KPI, filtres et pagination
  • saisie_mairie.php — Formulaire avec verrouillage pessimiste SQL
  • login.php — Connexion par sélection de poste (sans mot de passe)
  • mes_saisies.php / voir_saisie.php — Consultation des saisies
Base de données (4 tables) :
  • utilisateurs · communes · elus · journal_activite
CD Client Lourd · Desktop

Muni-Prospection — Version Desktop (Python)

Application desktop en Python 3.13 avec CustomTkinter 5.2 et architecture MVC (Modèle / Vue / Contrôleur dans un seul fichier main.py). Interface moderne avec header, barre de navigation et 3 onglets.

Python 3.13CustomTkinterMySQL Connectortkinter TreeviewMVC
Architecture MVC (main.py) :
  • Classe Modele — Requêtes SQL (CRUD communes, élus, saisies)
  • Classe Vue — Interface CustomTkinter (header, navbar, 3 onglets)
  • Classe Controleur — Logique métier, cache, export CSV
Les 3 onglets :
  • Dashboard — KPI en temps réel, statistiques
  • Validation — Validation des saisies des prospecteurs
  • Export — Export CSV des données filtrées

IA & Outils 2025 — 2026

OpenClaw : l'assistant IA open-source qui a explosé en 2026

OpenClaw est un assistant IA gratuit et open-source créé fin 2025. Il permet d'automatiser des tâches sur son ordinateur via des applis de messagerie comme Discord ou WhatsApp. Le projet a dépassé 350 000 stars sur GitHub en quelques mois. Un pote m'en a parlé et on a testé ensemble — c'est impressionnant pour automatiser des trucs répétitifs quand on code.
